# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |

## Reporting a Vulnerability

如果您发现安全漏洞，请**不要**通过公开的 Issue 报告。

### 报告方式

请通过以下方式之一私下报告安全问题：

1. 使用 GitHub 的 [私密安全建议功能](https://github.com/opzc35/Computer-resource-monitoring/security/advisories/new)
2. 发送邮件到项目维护者

### 报告内容应包括

- 漏洞的详细描述
- 重现步骤
- 潜在影响
- 可能的修复方案（如果有）

### 响应时间

- 我们会在 **48 小时内**确认收到您的报告
- 我们会在 **7 天内**提供初步评估
- 我们会尽快发布修复补丁

### 安全更新

安全更新将通过以下方式发布：

1. GitHub Releases
2. 安全公告
3. 版本更新说明

感谢您帮助保持项目的安全！
